# Agora · 方向调整：本地优先（Slock 模式）

> 2026-06 决策:**放弃"纯云多租户 SaaS",改走 Slock 式"本地优先 / 混合"路线。**
> 取代 [生产架构与路线图.md](生产架构与路线图.md) 的"多租户 SaaS"定位;该文档的技术细节(RLS/agent/RAG)仍适用,只是产品形态变了。

---

## 为什么转

想要"授权一下用 Claude 订阅、不配 key"那种丝滑体验 —— 它**只有本地执行才能做到**(用用户自己的算力/凭证)。纯云 SaaS 给不了,只能 BYO-key 或托管计费。Slock、vibe-kanban 都因此选了本地/混合,不是偶然。我们也跟。

## 新形态

**本地优先 / 混合**:
- **本地那半(现在做)**:agent 在用户自己机器上跑(用户的 key/订阅/算力)。聊天 UI + agent 引擎 + 存储都在本地。
- **云端那半(以后可选)**:协作/同步层(多人共享频道)。要做"混合"时再加,不是现在。

## 留什么 / 砍什么

| 留(核心,已建好,全部复用) | 砍 / 搁置(云 SaaS 专属) |
|---|---|
| agent 引擎(循环/工具/tool_runner) | ❌ 多租户 SaaS 野心 |
| 记忆 + RAG(embeddings/retrieval/pgvector) | ❌ Clerk 强鉴权(本地单用户不需要;DEV 模式本就脱离 Clerk 能跑) |
| 聊天 UI(Next.js) | ❌ Stripe 计费 |
| FastAPI 后端 + Postgres | ❌ "对外开放注册"的多租户运营 |

**数据层(org_id + RLS)处理**:不折腾、不删。本地按"单用户 / 单默认 org"用,RLS 留着无害(还省得以后要做混合云协作时重建)。

## 模型接入(本地)

- **现在**:本地存一个 API key(在设置页填一次,加密存本地库)。最简单稳健,保留我们自己的 agent 引擎。
- **以后可选**:加"授权 Claude/Codex 订阅"的本地 OAuth(像 codex/claude-code 弹链接登录)。
  - ⚠️ 注意:用订阅 OAuth + 我们自己的引擎,需要逆向 Claude 的 OAuth,**脆弱/灰色/可能违反条款**。要做得想清楚值不值;另一条是"编排现成 CLI"(白嫖授权但我们的引擎退居编排层)。这个等真要做时再单独决策。

## 路线图后段重排(替换原阶段 6/7 的计费/SaaS 上线)

阶段 0–3(对话/工具/记忆 RAG)✅ 已完成,与形态无关,继续有效。

- **阶段 4 · 多智能体协作**(引入 LangGraph)—— 与形态无关,照做。
- **阶段 5 · 可观测 + 成本核算** —— 照做(本地也想看 token 花了多少)。
- **阶段 6(新)· 本地打包与分发** —— 让别人能在自己机器一键起:桌面 app(Tauri/Electron)或 `npx`/docker 一键脚本。
- **阶段 7(新,可选)· 模型授权升级 + 混合云协作** —— 订阅 OAuth 选项;云端协作/同步层(真要做混合时)。

~~原阶段 6 计费 / 阶段 7 SaaS 上线~~ —— 本地优先下不需要,删。
